# スキル: 広告サービス申込書エクセル作成

## トリガー
- 「申込書」「申込書作って」「申込書エクセル」と言われたら実行

## 目的
DYMの「広告サービスに関する申込書」エクセルを、クライアントごとの情報を渡されたら
既存フォーマットを崩さずに埋めて新規ファイルとして出力する。パターン（広告種別）ごとに
セルの意味を覚えておき、指示されたパターンで即座に埋められるようにする。

## 動作原則
- **絶対に元のレイアウト・書式・数式・結合セルを変更しない**。値を書き込むだけ。
- 既存の申込書ファイル（テンプレートやサンプル）を読み込む際、Downloadsフォルダにある場合は
  ユーザーから明示的な指示があった場合のみアクセス可（CLAUDE.md参照）。
- 出力ファイルは `Pictures\Screenshots\商材資料\新しいフォルダー\提案資料\` に保存する。
  **`work\提案資料\`ではない**（2026-07-23、安藤さんが手動でこちらに移動済みと確認。フォルダ名は雑然としているが正しい保存先なので動かさない）
  ファイル名: `YYYYMMDD_株式会社◯◯御中_SNS広告御申込書.xlsx`
  （YYYYMMDDは**契約開始日ではなく作成日＝今日の日付**。末尾タグは元テンプレートのファイル名規則
  「SNS広告御申込書」に統一し、パターン名は入れない。2026-07-23確認済み）
- 完成したファイルはSendUserFileでユーザーに送り、クリックで開けるようにする。
- openpyxlで値を編集 → 可能なら`recalc.py`で数式を再計算。
  ※このPC(Windows)環境ではLibreOffice再計算スクリプトが動かない
  （`module 'socket' has no attribute 'AF_UNIX'`エラー）。動かない場合は
  手動で数式ロジックを確認し、Excelで開けば自動再計算される旨をユーザーに伝える。
- 曖昧・推測が必要な項目（契約終了日の計算方法、担当者TELを個人携帯にするか会社固定にするか等）は
  **絶対に憶測で埋めず、必ずユーザーに確認する**。

## 実行手順
1. Downloadsに新しいテンプレートがある場合はユーザーの明示指示があるか確認してから読む。なければ
   `work\提案資料\`内の過去出力ファイルや本スキルのセルマップだけで作業する。
2. ユーザーから渡された情報を上表にマッピング。
3. 不明点（特に契約期間の終了日計算、担当者TELの扱いなど）は必ず質問してから埋める。
4. openpyxlで値のみ書き込み、既存の書式・数式・結合セルは一切変更しない。
5. `work\提案資料\`に `YYYYMMDD_株式会社◯◯御中_SNS広告(パターン名)申込書.xlsx` で保存。
6. 保存後、書き込んだセルの値を読み返してユーザーに確認用サマリを出す。

### パターン: AIO対策 御見積書兼申込書（見積書シート）
- 別ファイル形式。ワークブックには複数シートが入っていることがあるが、**「見積書」というシート名
  （タイトルが「AIO対策における御見積書　兼　御申込書」になっている方）だけを使う**。
  他のシート（例:「見積兼申込書10万円」など古い固定SEOパターンの中身が残っているもの）は
  元々`state="hidden"`（非表示）で同梱されており、**別クライアントの機密情報が残っている**
  （2026-07-23、実際に無関係な別クライアントの契約情報が残っていたことがあった）。

- **重要: このファイルには埋め込みWordドキュメント・図形描画（xl/embeddings, xl/drawings, xl/media）が
  含まれている。openpyxlで読み込んで`del wb[sheet]`のようにシートを削除して保存すると、これらが
  失われてExcelで「修復」warningが出るほど壊れる（2026-07-23に実際に発生・要因を特定済み）。
  このテンプレートを編集する時はopenpyxlを使わず、zipfileでxlsxを開いて対象シートのXML
  （`xl/worksheets/sheetN.xml`）とsharedStrings.xmlだけをピンポイントで文字列置換し、
  他のzipエントリ（embeddings/drawings/vmlDrawing/printerSettings等）はバイト単位でそのままコピーする方式で行う。
  他クライアント情報が残る非表示シートも、削除すると同様に壊れるリスクがあるので、
  シート自体は残したまま該当セルの中身だけ同じXML直接編集方式で空白化する（シート削除はしない）。**

- **ハイパーリンク罠**: メール・URLのセル（例: C8, B12, C29）にはExcelのハイパーリンクが
  セル本体とは別に`xl/worksheets/_rels/sheetN.xml.rels`で紐づいている。セルの表示テキストだけ
  書き換えても、クリック時の飛び先（mailto:/URL）は古いクライアントのままになる
  （openpyxl経由でも同様の罠あり。cell.valueとhyperlink.targetは別物で、値を変えてもリンク先は自動で変わらない）。
  **表示テキストを変えたセルは、対応する.relsのTarget属性も必ず一緒に書き換える**
  （2026-07-23、SNS・SEO・AIOの3パターン全てで実際にこの不整合が起きて気づいた）。
